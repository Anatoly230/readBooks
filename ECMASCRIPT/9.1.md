# 9.1 Экологические рекорды

Environment Record — это тип спецификации, используемый для определения [](https://tc39.es/ecma262/#prod-Identifier)ассоциации идентификаторов с конкретными переменными и функциями, основанный на лексической структуре вложения кода ECMAScript. Обычно запись среды связана с какой-то специфической синтаксической структурой кода ECMAScript, такой как [FunctionDeclaration](https://tc39.es/ecma262/#prod-FunctionDeclaration), [BlockStatement](https://tc39.es/ecma262/#prod-BlockStatement) или [Catch](https://tc39.es/ecma262/#prod-Catch) clause [TryStatement](https://tc39.es/ecma262/#prod-TryStatement). Каждый раз, когда такой код оценивается, создаётся новая запись среды для записи привязок идентификаторов, созданных этим кодом.

В каждой записи среды есть поле, которое либо является null, либо является ссылкой на внешнюю запись окружающей среды. Это используется для моделирования логического вложения значений Environment Records. Внешний ориентир (внутреннего) экологического записи — это ссылка на запись окружающей среды, которая логически окружает внутреннюю запись окружающей среды. Открытый экологический запись, конечно, может иметь свой собственный внешний экологический рекорд. Запись окружающей среды может служить внешней средой для нескольких записей внутренней среды. Например, если [FunctionDeclaration](https://tc39.es/ecma262/#prod-FunctionDeclaration) содержит две вложенные [FunctionDeclaration](https://tc39.es/ecma262/#prod-FunctionDeclaration), то Окружающие записи каждой из вложенных функций будут иметь в качестве внешнего Environment Record Окружающую запись текущей оценки окружающей функции.[[OuterEnv]]

Записи среды — это исключительно механизмы спецификации и не обязательно соответствуют какому-либо конкретному артефакту реализации ECMAScript. Программа ECMAScript может напрямую получить доступ к таким значениям или манипулировать ими.
