>[!note]+ Пояснение
>let и const — это объявления переменных, область действия которых ограничена LexicalEnvironment текущего контекста выполнения. Переменные создаются при создании экземпляра содержащего их объекта Environment Record, но доступ к ним невозможен до тех пор, пока не будет вычислено LexicalBinding переменной. Переменной, определенной с помощью LexicalBinding с инициализатором, присваивается значение выражения AssignmentExpression инициализатора при вычислении LexicalBinding, а не при создании переменной. Если у LexicalBinding в объявлении let нет инициализатора, то при вычислении LexicalBinding переменной присваивается значение undefined.
