Когда мы говорим, что функции являются «родительскими классами», мы имеем в виду, что они такие же, как и все остальные... то есть, другими словами, обычные классы. Мы можем обращаться с функциями как с любым другим типом данных, и в них нет ничего особенного — их можно хранить в массивах, передавать в качестве параметров функций, назначать переменным и т. д.

Это базовые знания по `JavaScript`, но о них стоит упомянуть, поскольку быстрый поиск кода на `github` покажет коллективное уклонение от этой концепции или, возможно, ее повсеместное незнание. Давайте рассмотрим придуманный пример? Давайте.

```js
const hi = name => `Hi ${name}`;
const greeting = name => hi(name);
```

Здесь функциональная оболочка вокруг `hi` в greeting является полностью излишней. Почему? Потому что функции в JavaScript являются вызываемыми. Когда `hi` имеет `()` в конце, она запускается и возвращает значение. Когда этого нет, она просто возвращает функцию, хранящуюся в переменной. Чтобы убедиться в этом, посмотрите сами:

```js
hi; // name => `Hi ${name}`
hi("jonas"); // "Hi jonas"
```
